# Project Board Rules

This file mirrors the project-management scheme that will be enforced with GitHub Issues.

Principles:
1. Each meaningful work item is one issue.
2. Issues are grouped by topic and chapter.
3. Status is tracked with labels, not with free-form text.
4. A task moves through a small number of explicit states.
5. Completion requires verification evidence, not agent self-report.

Suggested labels:
- `area:repo`
- `area:docs`
- `area:experiments`
- `area:smoke`
- `area:tests`
- `area:infra`
- `chapter:01` ... `chapter:12`
- `status:ready`
- `status:in-progress`
- `status:blocked`
- `status:review`
- `status:done`
- `priority:P0` ... `priority:P3`

Issue groups:
- repo bootstrap
- chapter map and documentation
- deterministic verification experiments
- real-agent smoke test harness
- CI / automation
- reporting and cleanup
